# F5: Stack completo open source (alternativa a BaaS)

## Findings

### [1] Supabase self-hosted requires minimum 4GB RAM / 2 cores / 40GB SSD, recommended 8GB+ RAM / 4+ cores / 80GB+ SSD
- quote: "Minimum requirements for running all Supabase components, suitable for development and small to medium production workloads: RAM 4 GB, CPU 2 cores, Disk 40 GB SSD"
- url: https://supabase.com/docs/guides/self-hosting/docker
- source_type: primary
- published: 2026 (docs current)
- confidence: high

### [2] Supabase self-hosted stack comprises 8+ services: Kong, PostgREST, GoTrue (Auth), Realtime, Storage, imgproxy, Edge Functions, Supavisor, plus Postgres
- quote: "The API gateway (Kong) sits in front of 7 services: GoTrue, PostgREST, Realtime, Storage, pg_meta, Functions, and pg_graphql. All the services talk to a single Postgres instance."
- url: https://supabase.com/docs/guides/self-hosting/docker
- source_type: primary
- published: 2026
- confidence: high

### [3] Supabase self-hosted is community-supported only; user bears all responsibility for security, updates, backups, HA, and monitoring
- quote: "When you self-host, you are responsible for: Server provisioning and maintenance, Security hardening and keeping OS and services updated, Service configuration and management, Postgres database maintenance, High availability and scalability, Backups and disaster recovery, Monitoring and uptime"
- url: https://supabase.com/docs/guides/self-hosting
- source_type: primary
- published: 2026
- confidence: high

### [4] AppWrite self-hosted requires minimum 2 CPU cores, 4GB RAM, 2GB swap — same RAM as Supabase but half the CPU minimum
- quote: "Minimum requirements: 2 CPU cores, 4GB of RAM, 2GB of swap memory"
- url: https://appwrite.io/docs/advanced/self-hosting/installation
- source_type: primary
- published: 2026 (v1.9.5)
- confidence: high

### [5] AppWrite self-hosted uses Docker-based single-command install with a web-based setup wizard, supporting MongoDB or MariaDB as database backend
- quote: "docker run -it --rm --publish 20080:20080 --volume /var/run/docker.sock:/var/run/docker.sock --entrypoint='install' appwrite/appwrite:1.9.5"
- url: https://appwrite.io/docs/advanced/self-hosting/installation
- source_type: primary
- published: 2026
- confidence: high

### [6] AppWrite self-hosted runs ~15+ containers including workers (audits, webhooks, deletes, databases, builds, screenshots, certificates, executions, functions, mails, messaging, migrations), MongoDB, Redis, executor, browser, and console
- quote: "The docker-compose.yml shows approximately 18 containerized services including traefik, appwrite core, console, realtime, 11 workers, 2 tasks, MongoDB, Redis, openruntimes-executor, browser, and assistant"
- url: https://appwrite.io/docs/advanced/self-hosting/installation
- source_type: primary
- published: 2026
- confidence: high

### [7] Oracle Cloud Always Free tier offers 4 Arm-based Ampere A1 OCPUs with 24GB RAM total — sufficient for both Supabase (min 4GB) and AppWrite (min 4GB) self-hosted on a single VM
- quote: "The Always Free tier includes up to 4 Ampere A1 OCPUs and 24GB of memory, usable as one VM or up to 4 VMs"
- url: https://www.oracle.com/cloud/free/
- source_type: primary
- published: 2026
- confidence: high

### [8] Oracle Cloud Always Free includes 200GB block volume storage, 20GB object storage (Autonomous DB), and 10TB/month outbound data transfer — enough for MP3 metadata/files for 10-50 users
- quote: "Always Free includes 200 GB of block storage, 20 GB of object storage, and up to 10 TB per month of outbound data transfer"
- url: https://www.oracle.com/cloud/free/
- source_type: primary
- published: 2026
- confidence: high

### [9] Supabase Docker setup can be installed in under 15 minutes with a single curl command that generates all secrets and keys automatically
- quote: "The fastest and recommended way to self-host Supabase is to use Docker. It should take you less than 15 minutes to get up and running."
- url: https://supabase.com/docs/guides/self-hosting/docker
- source_type: primary
- published: 2026
- confidence: high

### [10] Supabase publishes stable Docker Compose releases approximately once a month; updates may cause downtime as services are stopped and restarted
- quote: "We publish stable releases of the Docker Compose setup approximately once a month. [...] After updating the configuration, stop and start the stack or individual services to pick up the changes. This may result in downtime for your applications and users."
- url: https://supabase.com/docs/guides/self-hosting/docker
- source_type: primary
- published: 2026
- confidence: high

### [11] AppWrite self-hosting explicitly warns that it often costs more than managed Cloud when factoring in time and expertise
- quote: "Consider self-hosting if you want to manage your own infrastructure [...] Appwrite Cloud is recommended if you expect self-hosting to save costs (it often costs more when factoring in time and expertise)"
- url: https://appwrite.io/docs/advanced/self-hosting
- source_type: primary
- published: 2026
- confidence: high

### [12] Both Supabase and AppWrite can fit within Oracle Cloud Always Free tier (4 Arm A1 cores, 24GB RAM), making $0/month infrastructure feasible for a 10-50 user MP3 downloader app
- Synthesis: Supabase needs min 4GB/2cores, AppWrite needs min 4GB/2cores. Oracle Free offers 24GB/4cores. Both stacks fit within a single Always Free VM with resources to spare.
- url: https://www.oracle.com/cloud/free/
- source_type: primary
- published: 2026
- confidence: high
